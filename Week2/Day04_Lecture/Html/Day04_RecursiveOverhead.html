Recursion - Overhead

Recursion is NOT equivalent to other kinds of loops. Every recursive method call generates an entry on the call stack. This generates a bit of memory overhead for every recursive call. If the recursive method continues making recursive calls for too long, the stack will run out of memory and cause a stack overflow exception! You should be careful about using recursion. It is not usually good for open-ended looping (like menus).

Two Loops

Recursion is also different from other loops in the sense that recursion has 2 loops. And often times, you'll do different things in the loops. 
Counter(0);

static void Counter(int N)
{
    if(N < 100)
    {
        Console.ForegroundColor = ConsoleColor.Blue;
        Console.WriteLine(N);

        Counter(N+1);

        //these lines only happen after N becomes 100
        Console.ForegroundColor = ConsoleColor.Red;
        Console.WriteLine(N);
    }
}
In this example, the numbers 0-99 are shown in blue. That is the first loop. However, when N becomes 100, the if condition becomes false and the Counter method can return. Where does a recursive method return to? The method returns to the method that called it. In this case, Counter(99) called Counter(100) so Counter(100) will return to where it was called. Then Counter(99) can finish the rest of its code and return to Counter(98). Etc. When the exit condition happens, it will then print 99-0 in red. This is the second loop sometimes called unwinding the stack since the method calls placed on the call stack start to be removed from the call stack.
