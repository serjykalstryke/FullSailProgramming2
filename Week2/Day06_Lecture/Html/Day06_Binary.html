Binary Search

The binary search algorithm is a fast algorithm.
  • It only works on sorted data
  • It divides-and-conquers. Each recursive call cuts the search list in half.
The algorithm (recursive):
  1. if the high index < the low index, return -1 (the item is not found)
  2. calculate the middle index using the low and high parameters
  3. if the search item == list[middle index], return the middle index
  4. else if the search item < list[middle index], recursively call BinarySearch for the left half of the list
  5. else if the search item > list[middle index], recursively call BinarySearch for the right half of the list
If the item is not found, -1 is returned because -1 is an invalid index and indicates to the calling code that the item was not found.

Performance

The worst-case performance for binary search is O(log N) - logarithmic. The binary search algorithm performs much better than the linear search algorithm, which is O(N) - linear.

Pseudocode for Binary Search

When calling Binary Search for the first time, pass 0 for low and N-1 for high, where N is the number of items in the list.

The algorithm ONLY works on sorted data. Therefore, the A parameter should be sorted prior to calling Binary Search

// initially called with low = 0, high = N-1. A is a sorted list.
BinarySearch(A[0..N-1], searchTerm, low, high) 
{
    if (high < low)
        return -1 // -1 means not found

    mid = (low + high) / 2

    if (searchTerm < A[mid])
        return BinarySearch(A, searchTerm, low, mid-1)
    else if (searchTerm > A[mid])
        return BinarySearch(A, searchTerm, mid+1, high)
    else
        return mid //the searchTerm was found
}
