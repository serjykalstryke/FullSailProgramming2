GitHub: Commit and Push

Commit and Push

When you're done working on your local code (or you simply want to back it up), you need to get the code into your GitHub repo. Currently, the code just resides on your local machine; it is NOT in GitHub.

You first commit your code then push it to GitHub.

1. Commit

To commit, you must have changed/added files in your local repo.

To open the Git Changes tab, go to Visual Studios View menu and select "Git Changes"

The commit is a 2-step process:
  1. Enter a message in the commit textbox.
  2. Click the "Commit All" button.

NOTE: if you've committed successfully, you'll see a message in the Git Changes tab that the commit was created locally. You'll also see that the outgoing number has increased. (the outgoing number can be seen below the branch dropdown)

2. Push

Once you've made a commit, press the push button (the up arrow) in the Git Changes tab.

NOTE: once the push is successful, you'll see a message in the Git Changes tab that you've successfully pushed. You'll also see that the outgoing number has been set back to 0. (the outgoing number can be seen below the branch dropdown)

If this step fails, check the Troubleshooting page for possible solutions.
