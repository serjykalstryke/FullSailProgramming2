Console: Converting

There are 2 ways to convert from one data type to another:

  • Implicit Conversion

    The compiler can convert from a smaller data type to a larger data type. No build warnings or errors will happen. 
    int smallNum = 5;
long bigNum = smallNum; //this is implicit conversion

  • Explicit Conversion

    The compiler can NOT convert from a larger data type to a smaller data type because there could be a loss of data in the conversion. Therefore, the compiler forces the developer to explicitly define the conversion.
    On the right-side of the assignment, put the type you want to convert to inside ( ). 
    long bigNum = 5;
int smallNum = (int)bigNum; //this is explicit conversion

Converting from strings to numeric types

You cannot convert a string to any numeric types with an implicit or explicit conversion. Neither will work.
For example, there are a couple of ways to convert a string to an int type:

  • int.Parse

    int.Parse will convert a string and attempt to convert to an integer.
    If it succeeds, the integer is returned.
    If it fails, an exception is thrown.

    To use int.Parse safely, you should use a try-catch to ensure that any exceptions are caught.

    string age = "Bob";
try {
    int myAge = int.Parse(age);
}
catch(Exception ex)
{
    Console.WriteLine($"{age} is not a number.");
}
    In this example, age is "Bob" so an exception is thrown and this is printed: "Bob is not a number."

  • int.TryParse

    int.TryParse will convert a string and attempt to convert to an integer.
    If it succeeds, the integer is returned through the out parameter and true is returned in the return type.
    If it fails, an exception is NOT thrown. A 0 is returned through the out parameter and false is returned in the return type. 
    string age = "Bob";
bool succeeds = int.TryParse(age, out int myAge);
    In this example, age is "Bob" so succeeds is set to false and myAge is set to 0.
