Arrays Pros/Cons

PROS

Indexer

One of the main advantages to using arrays is the speed of lookup using the indexer. int num = numbers[1];
The performance of the indexer is O(1) or order-of(1). That is often referred to as constant which means the performance does NOT change -- it is constant.
It doesn't matter if the array has 10 items or 10,000 items. It takes the same amount of time to use the indexer.

Statically sized data

Another pro of arrays is that arrays are good if the number of items does not need to change often.
For instance, if you always need 10 items and don't need to increase or decrease the number of items (statically sized), then an array is a good option.

CONS

Dynamically sized data

If you need to grow or shrink your array, then an array does not perform well.
You need to do the following:
  • Create a temp array with the new size
  • Copy the items to the temp array
  • Assign the temp array to the array variable
int[] temp = new int[7];
for(int i=0;i < nums.Length; i++)
    temp[i] = nums[i];
nums = temp;
The code above would get more complicated if you needed to insert the new item into the array or you were shrinking the array as the indexes would not match.
This is error prone and would cause a duplication of the data.
In addition, you would have to do this code every time the array changed size.
