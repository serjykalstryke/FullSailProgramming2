Methods Overview

Methods are where the real work in software happens! Variables are necessary but they just store data. The methods do the work and where you'll spend the majority of your career coding. Therefore, learning how to create methods and call them is vital to being a software engineer.

Methods are just named blocks of code. Therefore, you need to know the name of the method and how to talk (return type and parameters) to the method. These are the base parts of a method signature (or header): return type, method name, and parameter list.

Examples

This line of code calls the ResetColor method on the Console class. The method takes no parameters and does not return any data. 

Console.ResetColor();

This line of code calls the SetCursorPosition method on the Console class. The method takes 2 parameters and does not return any data.
It moves the cursor to the 5th column from the left and 10th row from the top. 

Console.SetCursorPosition(5, 10);

This line of code calls the ReadLine method on the Console class. The method takes no parameters and returns a string entered by the user. When a method returns data, usually you assign the returned value to a variable so you can use the data that is returned. 

string usersInput = Console.ReadLine();
