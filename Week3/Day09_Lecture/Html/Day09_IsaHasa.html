Is-A vs. Has-A

In general, you want to favor composition (has-a) over inheritance (is-a).

The inheritance hierarchy is built into your code and cannot change at runtime. Has-a (the items contained inside your class) can be pointed to different instances making your code more flexible.

When possible, program to an interface. Doing so will create looser coupling and make your code more flexible.
