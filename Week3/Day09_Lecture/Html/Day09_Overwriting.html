OOP - Overwriting

Overriding allows you to change the behavior of a base method in a derived class method.

Requirements

  • Mark the base class method with the virtual keyword.
  • Create a derived class method with the same signature but mark it with the override keyword.

Two ways to override

  • Extension

    If you want the behavior of the base method and add new functionality to it (extend it), then call the base version of the method from inside the derived method.
    base.Method(parameters);

    To call the base class method, make sure to put base. in front of the method call. If you do not, you will cause a recursive loop.

  • Fully Override

    If you do NOT want any of the behavior of the base method, simply do not call the base method.

Examples

Extend the base method

public class Employee
{
    //mark it with 'virtual'
    public virtual void DoWork(int hours) 
    {
        Console.WriteLine($"Doing {hours} hours of work today!!");
    }
}
public class Manager : Employee
{
    //mark it with 'override'
    public override void DoWork(int hours) 
    {
        base.DoWork(hours-4);//extend the method by calling the base
        Console.WriteLine($"I'll be in meetings the rest of the time. :-( ");
    }
}
If you remove this line from above (base.DoWork(hours-4);), you will be fully overriding the method.

Behavior

When you call the overriden method, the method that is called is determined by the object it is called on. If the object is just a base object, the base version is called. If the object is a derived object (even if the variable is a base type variable), the derived version is called. 
Employee employee = new Employee();
employee.DoWork(6); // will call the Employee class DoWork method

employee = new Manager();
employee.DoWork(8); // will call the Manager class DoWork method
