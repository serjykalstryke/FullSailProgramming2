OOP - Inheritance and Constructors

Every class needs a constructor to initialize the data in your class.

When you inherit from another class, your derived class must call a base constructor to ensure that the base class parts are initialized before the parts of the derived class are initialized.

To call a base constructor, you usually need to pass data to the base constructor. Therefore, the data needs to also be passed to the derived class constructor.

Examples

Calling the base constructor

After the last ) on the derived class constructor, add : base(parameters). Call it like a method (just pass the variables without the type information). 
public class Person
{
    public string Name {get;set;}
    //constructor
    public Person(string name) 
    { 
        Name = name; 
    }
}
public class Superhero : Person
{
    public string Identity {get;set;}
    public Superhero(string identity, string name) : base(name) //call the base constructor
    { 
        Identity = identity;
    }
}
The Superhero constructor needs to pass a string to the base constructor. Therefore, a string name parameter must be passed to the Superhero constructor. Then, it calls the base constructor with : base(name).

Do NOT duplicate the base constructor code inside of the derived constructor. Let the base constructor do its work when you call :base(). The code in the derived constructor should be what's special about the derived constructor and should only initialize the derived class parts.
