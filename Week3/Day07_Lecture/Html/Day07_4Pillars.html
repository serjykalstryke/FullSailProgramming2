OOP - 4 Pillars

Abstraction

Abstraction is the grouping of data (variables) and behavior (methods) to give them more meaning.

Example

By itself, this variable is just a color. There's no extra meaning given to it.
ConsoleColor color = ConsoleColor.Green;
Now, because the color variable is inside a class called Car, it represents not just a color but the color of the car. More meaning is attached to the variable because of its association with the class. 
class Car
{
    ConsoleColor color = ConsoleColor.Green;
}

Encapsulation

Encapsulation is also about grouping but this time it is about bringing common functionality together and to hide how something works. Code that uses your class does not need to know about all the methods and variables that compose the class. They just need access to the functionality provided by the class.
