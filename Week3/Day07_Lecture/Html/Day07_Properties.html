OOP - Properties

Properties are the gate-keepers of your fields. They are meant to control the access to fields.
Traditionally in languages like C++, this was handled with getter and setter methods.
In C#, getters and setters have been formalized into the language as properties.

Naming Convention

When naming C# properties, follow Pascal casing. Pascal casing starts every word in a compound word with a capital letter. Example: ThisIsACompoundWord

Access Modifiers

Usually you want your properties to be public. They are the public facing "data" of your class.

Get and Set

There are 2 parts to a property: the get method (accessor) and the set method (mutator). 
public class Car
{
    private int _yearOfCar;//hide it with private

    public int Year
    {
        get 
        {
            return _yearOfCar;
        }
        
        set 
        {
            if(value > 1908)
                _yearOfCar = value;
        }
    }
}
Get
The get provides access to the value of the field. It is the same as writing a getter method like this: 
public int GetYear() {return _yearOfCar;}
Set
The set allows modification of the field. It is the same as writing a setter method like this: 
public void SetYear(int value) {  if(value > 1908) _yearOfCar = value;  }
Usage
When using the property, the compiler will figure out whether to call the get or the set, depending on how it is being used. 
Car myCar = new Car();
myCar.Year = 1965; //this will call the set of the Year property
Console.WriteLine($"The year of your mustang is {myCar.Year}."); //this will call the get of the Year property

Full Properties

Full properties are properties that are paired with a backing field. In the examples above, _yearOfCar is the backing field to the Year property.

If you write any code inside the get or set, a backing field is required.

Auto Properties

Auto properties are properties where the backing field and code for the get and set are provided by the compiler. You only need to define the property.

Auto properties are good if you want to provide a light-weight property with differing access modifiers on the get/set

public class Car
{
    public string Make {get; private set;}
}
When creating an auto property, you don't write code for the get and set. In this example, the get is public but the set is private. So code outside the class can see the value but cannot change it. That is something you cannot do with a field.
