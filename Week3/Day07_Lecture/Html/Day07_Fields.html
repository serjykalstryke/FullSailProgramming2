OOP - Fields

Fields are the data (or variables) of your class. They are just like variables in your methods except their scope is at the class level instead of the method.

Access Modifiers

You generally want to keep your fields hidden from code outside of the class. Therefore, you would normally use the private access modifier on your fields.

private is the default accessibility given to items inside classes. So if it is not specified, private is applied.

Naming Convention

When naming C# fields, prefix the name with an underscore (_) and follow camel casing. Camel casing starts every word in a compound word with a capital letter except the first word. Example: thisIsACompoundWord

The reason we prefix with an _ is so developers can visually distinguish a field from a local variable. (_yearOfCar versus yearOfCar). Camel casing is to make it easier to read compound words. EX: thisIsEasierToRead versus thisishardertoread.

Static vs Instance Fields

If the static keyword is applied to a field, then the field is attached at the class level. The only way to access the static field is through the class name.
Instance fields are those fields without the static keyword. They are only visible on the instances of the class and can only be accessed through the instance.
Examples
public class Car
{
    public static int _numberOfCarsBuilt = 0;

    public int _yearOfCar;
}
_numberOfCarsBuilt is a static field and can be accessed outside of the class using the class name (Car). 
Console.WriteLine($"Cars built: {Car._numberOfCarsBuilt}");
_yearOfCar is an instance field and can be accessed outside of the class only with an instance of the Car class. 
Car myCar = new Car();
Console.WriteLine($"Year of Car: {myCar._yearOfCar}");
